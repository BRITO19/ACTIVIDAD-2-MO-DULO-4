ACTIVIDAD 2 MÓDULO 4 (Análisis y diseño de sistemas)

  1. Diseña un plan de proyecto en el que se definan las diferentes fases y actividades necesarias para desarrollar un sistema de gestión de inventario. Asocia cada actividad con su respectiva fase y establece las dependencias lógicas entre ellas.

    2. Fase 1: Iniciación

    Actividad 1.1: Definición del Alcance del Proyecto

    • Dependencias: Ninguna.
    • Descripción: Identificar y documentar las partes interesadas clave, sus necesidades y expectativas.
    • Entregable: Registro de Partes Interesadas.
    • Duración Estimada: 1 semana.

    Actividad 1.2: Identificación de Partes Interesadas

    • Dependencias: Completar Actividad 1.1.
    • Descripción: Definir claramente los objetivos y el alcance del proyecto, incluyendo las funcionalidades y los límites del sistema de gestión de inventario.
    • Entregable: Documento de Alcance del Proyecto.
    • Duración Estimada: 1 semana.

    Actividad 1.3: Desarrollo del Caso de Negocio

    • Dependencias: Completar Actividad 1.1.
    • Descripción: Elaborar un caso de negocio que justifique el proyecto, incluyendo beneficios, costos y riesgos.
    • Entregable: Caso de Negocio.
    • Duración Estimada: 2 semanas.

    Actividad 1.4: Aprobación del Proyecto

    • Dependencias: Completar Actividades 1.1, 1.2 y 1.3
    • Descripción: Obtener la aprobación formal del proyecto por parte de los patrocinadores y partes interesadas.
    • Entregable: Aprobación Formal del Proyecto.
    • Duración Estimada: 1 semana.

    Fase 2: Planificación

    Actividad 2.1: Desarrollo del Plan de Proyecto

    • Dependencias: Completar Actividad 1.4.
    • Descripción: Crear un plan detallado que incluya cronograma, recursos, presupuesto, y estrategias de gestión de riesgos.
    • Entregable: Plan de Proyecto.
    • Duración Estimada: 2 semanas.

    Actividad 2.2: Recolección de Requisitos

    • Dependencias: Completar Actividad 1.4.
    • Descripción: Recopilar requisitos detallados a través de entrevistas, encuestas, y análisis de procesos existentes.
    • Entregable: Documento de Requisitos.
    • Duración Estimada: 3 semanas.

    Actividad 2.3: Análisis de Riesgos

    • Dependencias: Completar Actividad 2.1.
    • Descripción: Identificar, evaluar y planificar la gestión de riesgos potenciales del proyecto.
    • Entregable: Registro de Riesgos.
    • Duración Estimada: 1 semana.

    Actividad 2.4: Planificación de Recursos

    • Dependencias: Completar Actividad 2.1.
    • Descripción: Definir y asignar los recursos necesarios, incluyendo equipo, herramientas y tecnología.
    • Entregable: Plan de Recursos.
    • Duración Estimada: 1 semana.

    Fase 3: Diseño

    Actividad 3.1: Diseño de la Arquitectura del Sistema

    • Dependencias: Completar Actividad 2.2.
    • Descripción: Crear la estructura técnica del sistema, incluyendo la arquitectura de software y hardware.
    • Entregable: Documento de Diseño de Arquitectura
    • Duración Estimada: 3 semanas.

    Actividad 3.2: Diseño de la Base de Datos

    • Dependencias: Completar Actividad 3.1.
    • Descripción: Diseñar el esquema de la base de datos para almacenar la información del inventario.
    • Entregable: Esquema de Base de Datos.
    • Duración Estimada: 2 semanas.

    Actividad 3.3: Diseño de la Interfaz de Usuario (UI)

    • Dependencias: Completar Actividad 3.1.
    • Descripción: Crear wireframes y prototipos de la interfaz de usuario para asegurar una experiencia de usuario óptima.
    • Entregable: Prototipos de UI.
    • Duración Estimada: 2 semanas.

    Actividad 3.4: Revisión y Aprobación del Diseño

    • Dependencias: Completar Actividades 3.1, 3.2 y 3.3.
    • Descripción: Revisar los diseños propuestos y obtener la aprobación de las partes interesadas.
    • Entregable: Aprobación del Diseño.
    • Duración Estimada: 1 semana.

    Fase 4: Desarrollo

    Actividad 4.1: Configuración del Entorno de Desarrollo

    • Dependencias: Completar Actividad 3.4.
    • Descripción: Preparar el entorno técnico necesario para el desarrollo del sistema.
    • Entregable: Entorno de Desarrollo Configurado.
    • Duración Estimada: 1 semanas.

    Actividad 4.2: Desarrollo del Backend

    • Dependencias: Completar Actividad 4.1.
    • Descripción: Implementar la lógica de negocio y la integración con la base de datos
    • Entregable: Código Backend.
    • Duración Estimada: 4 semanas.

    Actividad 4.3: Desarrollo del Frontend

    • Dependencias: Completar Actividad 4.1.
    • Descripción: Implementar la interfaz de usuario siguiendo los diseños aprobados.
    • Entregable: Código Frontend.
    • Duración Estimada: 4 semana.

    Actividad 4.4: Integración del Sistema

    • Dependencias: Completar Actividades 4.2 y 4.3.
    • Descripción: Integrar el frontend y el backend para asegurar que funcionen conjuntamente.
    • Entregable: Sistema Integrado.
    • Duración Estimada: 2 semana.

    Actividad 4.5: Pruebas de Unidad

    • Dependencias: Completar Actividades 4.2 y 4.3.
    • Descripción: Realizar pruebas de componentes individuales del sistema para verificar su correcto funcionamiento.
    • Entregable: Informes de Pruebas de Unidad.
    • Duración Estimada: 2 semanas.

    Fase 5: Pruebas

    Actividad 5.1: Pruebas de Integración

    • Dependencias: Completar Actividad 4.4.
    • Descripción: Verificar que los componentes del sistema se integren adecuadamente y funcionen como un todo.
    • Entregable: Informes de Pruebas de Integración.
    • Duración Estimada: 2 semana.

    Actividad 5.2: Pruebas de Usuario (UAT)

    • Dependencias: Completar Actividad 5.1.
    • Descripción: Permitir que los usuarios finales prueben el sistema y proporcionen retroalimentación.
    • Entregable: Informes de Pruebas de Usuario.
    • Duración Estimada: 2 semana.

    Actividad 5.3: Resolución de Defectos

    • Dependencias: Completar Actividades 5.1 y 5.2.
    • Descripción: Corregir los errores identificados durante las pruebas.
    • Entregable: Código Corregido.
    • Duración Estimada: 2 semanas.

    Fase 6: Implementación

    Actividad 6.1: Preparación del Entorno de Producción

    • Dependencias: Completar Actividad 5.3.
    • Descripción: Configurar el entorno de producción para el despliegue del sistema.
    • Entregable: Entorno de Producción Configurado.
    • Duración Estimada: 1 semana.

    Actividad 6.2: Despliegue del Sistema

    • Dependencias: Completar Actividad 6.1.
    • Descripción: Implementar el sistema en el entorno de producción.
    • Entregable: Sistema en Producción.
    • Duración Estimada: 1 semanas.

    Actividad 6.3: Capacitación de Usuarios

    • Dependencias: Completar Actividad 6.2.
    • Descripción: Entrenar a los usuarios finales sobre el uso del sistema.
    • Entregable: Materiales de Capacitación y Registros de Entrenamiento.
    • Duración Estimada: 1 semanas.

    Actividad 6.4: Monitoreo Post-Implementación

    • Dependencias: Completar Actividad 6.2.
    • Descripción: Supervisar el sistema en producción para identificar y resolver problemas.
    • Entregable: Informes de Monitoreo.
    • Duración Estimada: 2 semana.

    Fase 7: Cierre

    Actividad 7.1: Revisión Final del Proyecto

    • Dependencias: Completar Actividad 6.4.
    • Descripción: Evaluar el proyecto para asegurarse de que se hayan cumplido todos los objetivos.
    • Entregable: Informe de Revisión Final.
    • Duración Estimada: 1 semanas.

    Actividad 7.2: Documentación Final

    • Dependencias: Completar Actividad 7.1.
    • Descripción: Preparar y entregar toda la documentación final del proyecto.
    • Entregable: Documentación Final del Proyecto.
    • Duración Estimada: 1 semanas.

    Actividad 7.3: Entrega Formal y Cierre del Proyecto

    • Dependencias: Completar Actividad 7.2.
    • Descripción: Formalizar la entrega del sistema a los patrocinadores y cerrar el proyecto.
    • Entregable: Acta de Cierre del Proyecto
    • Duración Estimada: 1 semanas.
